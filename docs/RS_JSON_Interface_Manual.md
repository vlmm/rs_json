За реализиране на RS485 master-multi slave JSON интерфейс, е необходимо да се вземат предвид следните ключови аспекти:

### 1. Архитектура на интерфейса
- **Master-Slave архитектура**: В системата ще има един Master (главно устройство) и множество Slave устройства (подчинени устройства). Master устройството контролира комуникацията и инициира съобщенията, докато Slave устройствата отговарят на запитванията.

### 2. Формат на съобщенията
- **JSON формат**: Съобщенията между Master и Slave устройства ще бъдат форматирани в JSON. Примерен формат на съобщение:
  ```json
  {
    "address": "slave_address",
    "command": "command_type",
    "data": {
      "key": "value"
    }
  }
  XX
  ```
  където `XX` е контролната сума, изчислена на базата на JSON частта.

### 3. Контролна сума
- **Контролна сума**: Всяко съобщение трябва да завършва с контролна сума, която се изчислява на базата на съдържанието на JSON частта. Например, може да се използва прост алгоритъм за сумиране на ASCII стойностите на символите в JSON частта и след това да се вземе остатъка от делението на 256.

### 4. Откриване на подчинените устройства
- **Процедура за откриване**: Master устройството може да инициира команда за откриване на подчинените устройства, която да изпрати запитване до всички адреси в мрежата. Всеки Slave, който получи запитването, трябва да отговори с уникален идентификатор.

### 5. Пинг команда
- **Пинг команда**: Master устройството може да изпрати пинг команда до конкретно Slave устройство, за да провери неговата наличност. Формат на пинг командата:
  ```json
  {
    "address": "slave_address",
    "command": "ping"
  }
  XX
  ```
  където `XX` е контролната сума, изчислена на базата на JSON частта.

### 6. Адресиране на устройства
- **Индивидуално и групово адресиране**: Всяко Slave устройство ще има уникален адрес. Master устройството може да изпраща съобщения до конкретно устройство, като посочи неговия адрес, или до всички устройства, като използва специален адрес (например "broadcast").

### 7. Примерен работен поток
1. **Инициализация**: Master устройството инициализира RS485 комуникацията.
2. **Откриване на устройства**: Master изпраща команда за откриване и получава отговори от Slave устройствата.
3. **Пинг**: Master изпраща пинг команда до конкретно Slave устройство, за да провери наличността.
4. **Изпращане на команди**: Master изпраща команди до конкретни или всички Slave устройства.
5. **Получаване на отговори**: Slave устройствата обработват командите и изпращат отговори обратно на Master.

### 8. Примерен код
Примерен код за изпращане на съобщение от Master до Slave:
```python
import json
import serial

def calculate_checksum(message):
    return sum(message.encode('utf-8')) % 256

def send_message(address, command, data):
    message = {
        "address": address,
        "command": command,
        "data": data
    }
    message_str = json.dumps(message)
    checksum = calculate_checksum(message_str)
    message_str += f"{checksum:02d}"  # Форматиране на контролната сума с две цифри
    
    # Изпращане на съобщението по RS485
    ser = serial.Serial('/dev/ttyUSB0', 9600)
    ser.write(message_str.encode('utf-8'))
    ser.close()

# Пример за изпращане на команда
send_message("slave_1", "set_value", {"value": 10})
```
